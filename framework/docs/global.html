f5_poll_until_all_tests_pass(next, err, curr) → {Undefined}

Implements a polling mechanism in Postman/Collection Runner/Newman.

When using the external delay mechanism (_f5_poll_apiurl) this function expects a Item in the Collection named '_F5_POLL_DELAY'. This Item will be shimmed into the execution flow and that Item will them callback into the {curr} item to retry.

If _f5_poll_max_tries is reached then '[Poller] Max Tries Reached' test will be marked as FAIL. If _f5_poll_bypass_timeout is NOT set the next request will be set to NULL to stop execution, otherwise execution will continue.

Parameters:
Name Type Description
next String

The Item in the Postman Collection to execute once all tests pass
err Object

Object to be passed to f5_check_response to test for unrecoverable error conditions.
curr String

[Optional] The name of the current Item. Normally auto-populated with current Item name
Properties:
Name Type Description
_f5_poll_max_tries Number

The max number of polls
_f5_poll_wait Number

The time in seconds to wait between polls
_f5_poll_useinternal Boolean

Use the internal while() loop to sleep (WARNING: this will block the thread)
_f5_poll_apiurl String

The URL for an API endpoint that implements a delay
_f5_poll_bypass_timeout Boolean

Bypass a poller timeout and continue instead of exit
_f5_poll_interator Number

The current iterator value
_f5_poll_curr String

Name of the current Item in the Collection
